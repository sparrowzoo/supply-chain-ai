# 三大智能工程引擎：Harness Engine、Context Engine、Prompt Engine

## 1. Harness Engine（测试桩/自动化引擎）

**定义：**
Harness Engine 负责搭建自动化测试环境（test harness），实现代码的持续验证、集成测试和回归测试。它可以是测试框架、CI/CD 工具链，也可以是企业级 DevOps 平台。

**核心作用：**
- 先写测试再写实现（TDD），保障需求落地。
- 每次变更自动运行测试，第一时间发现缺陷。
- 支持自动回滚、灰度发布，提升系统稳定性。
- 测试用例即活文档，便于知识传承。

**典型工具：** pytest、JUnit、GitHub Actions、Harness（DevOps 平台）等。

---

## 2. Context Engine（上下文引擎）

**定义：**
Context Engine 用高度提炼、结构化的语言，描述项目的元数据和业务背景，让 AI 能像“同事”一样理解你的系统。

**核心内容：**
- 你是谁（团队/个人背景、技术栈、业务领域）
- 项目是做什么的（目标、核心业务、关键场景）
- 代码结构地图（模块、目录、依赖关系）
- 当前你要干什么（开发/重构/调试目标与边界）
- 约束与规范（命名、架构、业务边界、技术约束）

**作用：**
- 让 AI 生成的代码更懂业务、更贴合实际。
- 降低沟通成本，提升协作效率。

**典型落地：** .copilot-instructions.md、MCP Server、知识图谱、团队文档等。

---

## 3. Prompt Engine（提示词引擎）

**定义：**
Prompt Engine 专注于设计、优化与大模型交互的提示词（Prompt），让 AI 更好地理解你的意图，生成高质量输出。

**核心要素：**
- 领域知识和业务理解是 Prompt 的本质。
- 技巧只是辅助，核心是“把需求说清楚、说专业”。
- 团队可沉淀高复用的 Prompt 模板，提升协作效率。

**作用：**
- 让 AI 输出更精准、更高效。
- 支持团队级知识复用和最佳实践沉淀。

**典型落地：** Prompt 模板库、A/B 测试、团队级 prompt 规范。

---

## 总结

Harness Engine 保障代码质量和交付安全，Context Engine 让 AI 真正“懂你”，Prompt Engine 让人机协作更高效。三者协同，是 AI 时代智能开发团队的核心竞争力。
