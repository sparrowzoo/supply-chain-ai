## fileLinkification
## 文件链接格式

When mentioning files or line numbers, always convert them to markdown links using workspace-relative paths and 1-based line numbers.
提及文件或行号时，始终使用工作区相对路径和从 1 开始的行号，转换为 markdown 链接。

NO BACKTICKS ANYWHERE:
禁止任何地方使用反引号：
- Never wrap file names, paths, or links in backticks.
	文件名、路径或链接绝不能用反引号包裹。
- Never use inline-code formatting for any file reference.
	文件引用绝不能用行内代码格式。

REQUIRED FORMATS:
必须使用的格式：
- File: [path/file.ts](path/file.ts)
	文件： [path/file.ts](path/file.ts)
- Line: [file.ts](file.ts#L10)
	行号： [file.ts](file.ts#L10)
- Range: [file.ts](file.ts#L10-L12)
	区间： [file.ts](file.ts#L10-L12)

PATH RULES:
路径规则：
- Without line numbers: Display text must match the target path.
	无行号时，显示文本必须与目标路径一致。
- With line numbers: Display text can be either the path or descriptive text.
	有行号时，显示文本可为路径或描述性文本。
- Use '/' only; strip drive letters and external folders.
	仅使用 '/'，去除盘符和外部文件夹。
- Do not use these URI schemes: file://, vscode://
	禁止使用 file://、vscode:// 等 URI 协议。
- Encode spaces only in the target (My File.md → My%20File.md).
	仅在目标中对空格编码（My File.md → My%20File.md）。
- Non-contiguous lines require separate links. NEVER use comma-separated line references like #L10-L12, L20.
	非连续行需分别链接，绝不能用逗号分隔的行号引用（如 #L10-L12, L20）。
- Valid formats: [file.ts](file.ts#L10) only. Invalid: ([file.ts#L10]) or [file.ts](file.ts)#L10
	仅允许 [file.ts](file.ts#L10) 这种格式，禁止 ([file.ts#L10]) 或 [file.ts](file.ts)#L10。
- Only create links for files that exist in the workspace. Do not link to files you are suggesting to create or that do not exist yet.
	只为工作区已存在的文件创建链接，不要为建议创建或尚不存在的文件生成链接。

USAGE EXAMPLES:
用法示例：
- With path as display: The handler is in [src/handler.ts](src/handler.ts#L10).
	路径作为显示文本：The handler is in [src/handler.ts](src/handler.ts#L10)。
- With descriptive text: The [widget initialization](src/widget.ts#L321) runs on startup.
	描述性文本：The [widget initialization](src/widget.ts#L321) runs on startup。
- Bullet list: [Init widget](src/widget.ts#L321)
	列表项：[Init widget](src/widget.ts#L321)
- File only: See [src/config.ts](src/config.ts) for settings.
	仅文件：See [src/config.ts](src/config.ts) for settings。

FORBIDDEN (NEVER OUTPUT):
禁止输出（绝不允许）：
- Inline code: `file.ts`, `src/file.ts`, `L86`.
	行内代码：`file.ts`、`src/file.ts`、`L86`。
- Plain text file names: file.ts, chatService.ts.
	纯文本文件名：file.ts、chatService.ts。
- References without links when mentioning specific file locations.
	提及具体文件位置时未加链接。
- Specific line citations without links ("Line 86", "at line 86", "on line 25").
	未加链接的具体行号引用（如 "Line 86"、"at line 86"、"on line 25"）。
- Combining multiple line references in one link: [file.ts#L10-L12, L20](file.ts#L10-L12, L20)
	多个行号合并为一个链接（如 [file.ts#L10-L12, L20](file.ts#L10-L12, L20)）。
