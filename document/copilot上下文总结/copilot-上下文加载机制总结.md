# Copilot 指令/配置文件与记忆文件自动加载机制总结

在 Copilot 智能体系统中，不同类型的文件在上下文加载策略上有明显区别，具体如下：

## 1. 指令/配置文件（如 copilot-instructions.md、AGENTS.md 等）
- 作用：定义 Copilot 的行为规范、可用 agent、代码风格、适用范围等。
- 加载方式：**每次对话或操作时自动加载**，确保 Copilot 始终遵循团队/个人设定的规则。
- 目的：保证行为一致性和可控性。

## 2. 记忆/参考文件（如 project-overview.md 等）
- 作用：存储项目结构、历史、概览等信息，供 Copilot 或用户按需查阅。
- 加载方式：**不会自动加载**，只有在用户主动请求或 agent 需要时才读取。
- 目的：节省上下文窗口空间，避免无关信息干扰，提高效率。

## 3. 设计原因
- 指令/配置文件内容直接影响 Copilot 行为，必须始终生效。
- 记忆/参考文件内容体量大且不总是相关，按需加载更灵活高效。
- 这样设计兼顾了“可控性”与“高效性”，提升了智能体系统的实用性和响应速度。

---

如需自定义 agent 或调整加载策略，可编辑 AGENTS.md、copilot-instructions.md 等配置文件。
