# 元数据说明

本说明文档梳理了 Copilot/Agent 请求过程中的主要元数据字段及工具能力，便于理解系统行为和排查问题。

---

## 一、请求元数据字段说明

| 字段名              | 含义与说明 |
|---------------------|------------|
| requestType         | 请求类型（如 ChatCompletions，表示对话补全） |
| model               | 使用的模型名称（如 gpt-4.1） |
| maxPromptTokens     | 单次请求最大输入 token 数（如 111616） |
| maxResponseTokens   | 单次请求最大输出 token 数（如 16384） |
| location            | 服务器或节点编号/位置标识 |
| otherOptions        | 其他请求参数（如 temperature、stream 等） |
| intent              | 业务意图。系统自动推断（如未定义则为 undefined），普通用户无法直接指定，只能通过明确的提示词表达需求来影响意图识别。|
| startTime           | 请求发起时间（ISO 格式） |
| endTime             | 请求结束时间（ISO 格式） |
| duration            | 请求总耗时（毫秒） |
| ourRequestId        | 本地请求唯一标识符 |
| requestId           | 请求唯一标识符（与上同） |
| serverRequestId     | 服务端请求唯一标识符 |
| timeToFirstToken    | 首 token 响应耗时（毫秒） |
| resolved model      | 实际分配的模型版本 |
| usage               | token 使用统计（见下表） |

### intent 字段补充说明

intent 字段用于标识本次请求的业务意图（如“代码生成”“单元测试”“文档总结”等）。

- Copilot/Agent 会根据你的输入内容、上下文和历史自动推断 intent。
- 普通用户无法在界面或参数中直接指定 intent 字段。
- **唯一能影响 intent 的方式，是通过更明确、具体的提示词表达你的需求。**
	- 例如：“请帮我生成单元测试代码”、“请总结以下文档内容”等。
- 开发者在调用 API 或自定义 Agent 时，可在 payload 里设置 intent 字段。

---

### usage 字段结构
| 字段名                    | 含义 |
|---------------------------|------|
| completion_tokens         | 本次 AI 回复生成的 token 数 |
| completion_tokens_details | 生成 token 细节（如被接受/拒绝的 token） |
| prompt_tokens             | 输入 prompt（含历史上下文）token 数 |
| prompt_tokens_details     | prompt 细节（如缓存命中 token 数） |
| total_tokens              | 总 token 数（输入+输出） |
| reasoning_tokens          | 推理相关 token（部分高级用法） |

## 二、可用工具（tools）分类与详细说明

下表对 Copilot/Agent 支持的 89 个工具进行了分类，并补充了主要功能说明：

### 1. 文件与目录操作
| 工具名 | 主要功能 |
|--------|----------|
| apply_patch | 应用代码补丁，自动修改文件（支持多段差异） |
| create_directory | 创建新目录，支持递归创建多级目录 |
| create_file | 新建文件并写入内容，自动创建所需目录 |
| list_dir | 列出指定目录下的所有文件和子目录 |
| read_file | 读取文件内容，支持指定行范围 |
| insert_edit_into_file | 插入或编辑文件内容，适合小范围修改 |

### 2. Jupyter Notebook 相关
| 工具名 | 主要功能 |
|--------|----------|
| create_new_jupyter_notebook | 新建 Jupyter Notebook 文件 |
| edit_notebook_file | 编辑 Notebook 某个 cell（支持插入、编辑、删除） |
| copilot_getNotebookSummary | 获取 Notebook cell 概览（类型、执行状态等） |
| read_notebook_cell_output | 读取 Notebook cell 的输出内容 |
| run_notebook_cell | 运行 Notebook 指定 cell |
| configure_notebook | 配置 Notebook 内核（如选择 Python 内核） |

### 3. 搜索与分析
| 工具名 | 主要功能 |
|--------|----------|

| semantic_search | 语义搜索代码、注释或文档内容 |

### 4. 终端与命令执行
| 工具名 | 主要功能 |
|--------|----------|
| run_in_terminal | 在终端执行 shell 命令，支持前后台运行 |
